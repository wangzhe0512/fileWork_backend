## 问题描述

当前 `CompanyTemplateService.saveReverseResult()` 方法在保存反向生成的子模板记录时存在两个问题：

1. **允许多个 active 状态模板共存**：方法注释说明"不自动归档旧模板（允许多个 active，前端手动切换）"，但实际业务需求要求一个企业下同时只有一个 active 状态的子模板。

2. **updated_at 字段为 NULL**：创建新模板时只设置了 created_at，未设置 updated_at，导致数据库中该字段为 NULL。

## 期望修复

- 创建新 active 模板时，自动将该企业其他 active 状态模板归档
- 创建新模板时同时设置 updated_at 字段（与 created_at 相同值）

## 技术栈

- Java + Spring Boot
- MyBatis-Plus ORM 框架
- MySQL 数据库

## 修改方案

### 修改目标文件

`src/main/java/com/fileproc/template/service/CompanyTemplateService.java`

### 修改内容

1. **在 `saveReverseResult` 方法中新增自动归档逻辑**：

- 在插入新模板前，查询该企业下所有 status = "active" 的其他子模板
- 将这些模板状态改为 "archived" 并更新 updated_at 时间

2. **设置 updated_at 字段**：

- 新模板创建时同时设置 `updatedAt` 字段，保持与 `createdAt` 相同

### 实现参考

可以复用 `setActive` 方法中现有的归档逻辑代码模式，确保代码一致性。
