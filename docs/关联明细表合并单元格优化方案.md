# 关联明细表合并单元格占位符优化方案

## 背景

关联采购明细和关联销售明细是4列表格（第1列分组标题，第2-4列数据），当前设计中data行第一列使用`{{_col__group_title}}`占位符，导致语义混乱且不符合数据与展示分离原则。

## 核心思路

**利用Word合并单元格的自然特性：**
- data行和subtotal行的第一列**不写占位符**（保持空或原始文本）
- 通过Word表格的合并单元格特性，第一列自动继承上一行的内容
- 只有group行第一列需要`{{_col__group_title}}`来显示分组名

## 技术要点

### Word合并单元格机制

当Word表格中第一列的多个单元格被合并时：
- 第一个单元格的内容会跨越多行显示
- 后续的单元格（在合并范围内）即使为空，也会显示为合并单元格的一部分
- **关键：不需要显式填充后续行的第一列**

### 数据流

```
Excel解析 → 数据提取 → 填充引擎 → Word渲染
   ↓           ↓           ↓          ↓
A列合并    group行:      data行:    第一列显示
"境外关联  _group_title  不填充     "境外关联
 采购"     ="境外关联    第一列      采购"
           data行:      →         (合并效果)
           不设置       跳过
           _group_title 第一列
```

## 风险评估

### 风险1：其他TABLE_ROW_TEMPLATE表格受影响
**缓解措施：** 使用`isSpecialTable()`判断，只对关联采购明细和关联销售明细做特殊处理

### 风险2：Word合并单元格识别失败
**缓解措施：** 子模板生成时保持原有的合并单元格设置（`<vMerge val="restart"/>`和`<vMerge/>`）

### 风险3：subtotal行第一列显示异常
**缓解措施：** subtotal行第一列也跳过填充，依赖合并单元格特性
