## 数据架构说明

### 核心概念

企业子模板的模块和占位符**独立于系统标准模板**，原因：
- 反向生成的子模板可能来自非标准模板的历史报告
- 每个子模板保留自己的模块和占位符结构，支持灵活定制

### 数据表关系

```
company_template (企业子模板)
    └── id, company_id, name, year, ...
    
    ↓ 一对多
    
company_template_module (企业子模板模块) [需要新建]
    └── id, template_id, name, code, description, sort
    
    ↓ 一对多
    
company_template_placeholder (企业子模板占位符) [需要扩展]
    └── id, module_id, template_id, 
        placeholder_name, name, type, 
        data_source, source_sheet, source_field, description,
        status, confirmed_type, paragraph_index, run_index, offset, ...
```

### 表结构详情

#### 1. company_template_module 表（需要新建）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | varchar(64) | 主键 |
| template_id | varchar(64) | 所属子模板ID，外键 |
| name | varchar(255) | 模块名称 |
| code | varchar(100) | 模块代码 |
| description | text | 模块描述（可选） |
| sort | int | 排序序号，默认0 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

#### 2. company_template_placeholder 表（需要扩展）

在现有表基础上，**新增以下字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| module_id | varchar(64) | 所属模块ID，外键（新增） |
| name | varchar(255) | 显示名称（新增） |
| type | varchar(20) | 类型：text/table/chart/image/ignore（新增） |
| data_source | varchar(255) | 数据源（新增） |
| source_sheet | varchar(255) | 来源Sheet（新增） |
| source_field | varchar(255) | 来源字段（新增） |
| description | text | 说明（新增） |

**现有字段保留**：
- id, template_id, placeholder_name
- status (confirmed/uncertain)
- confirmed_type
- paragraph_index, run_index, offset
- context
- created_at, updated_at

## 接口清单

| 序号 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 1 | `/company-template/{templateId}/modules` | GET | 获取子模板模块列表 |
| 2 | `/company-template/{templateId}/modules/{moduleId}/placeholders` | GET | 获取模块占位符列表 |
| 3 | `/company-template/{templateId}/placeholders/{placeholderId}` | PUT | 更新占位符信息 |
| 4 | `/company-template/sync-placeholders` | POST | 批量同步占位符到其他子模板 |

## 1. 获取子模板模块列表

### 接口信息
- **URL**: `/api/company-template/{templateId}/modules`
- **Method**: GET
- **Content-Type**: application/json

### 请求参数

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|--------|------|------|------|------|
| templateId | string | path | 是 | 子模板ID |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": "module-001",
      "templateId": "template-001",
      "name": "基本信息",
      "code": "basic_info",
      "description": "企业基本信息模块",
      "sort": 1
    },
    {
      "id": "module-002",
      "templateId": "template-001",
      "name": "财务数据",
      "code": "financial_data",
      "description": "财务相关数据模块",
      "sort": 2
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 模块ID |
| templateId | string | 所属子模板ID |
| name | string | 模块名称 |
| code | string | 模块代码 |
| description | string | 模块描述（可选） |
| sort | number | 排序序号 |

**数据来源**：`company_template_module` 表，按 `template_id` 查询，按 `sort` 排序

## 4. 批量同步占位符到其他子模板

### 接口信息
- **URL**: `/api/company-template/sync-placeholders`
- **Method**: POST
- **Content-Type**: application/json

### 业务说明

将源子模板中选定的占位符同步到目标子模板。

**同步规则**：
1. 只同步占位符的元数据（name、type、dataSource、sourceSheet、sourceField、description）
2. 目标子模板中必须有相同的 `placeholder_name` 且属于同一 `module_code`（或相同模块名称）才进行同步
3. 只同步到**同一企业**下的其他子模板（安全检查）
4. 不创建新的占位符，只更新已存在的
5. 同步时 `status` 和 `confirmedType` 字段保持不变

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sourceTemplateId | string | 是 | 源子模板ID |
| targetTemplateIds | string[] | 是 | 目标子模板ID列表 |
| placeholderIds | string[] | 是 | 要同步的占位符ID列表 |

### 请求示例

```json
{
  "sourceTemplateId": "template-2024",
  "targetTemplateIds": ["template-2023", "template-2025"],
  "placeholderIds": ["ph-001", "ph-002"]
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "success": true,
    "syncedCount": 4,
    "details": [
      {
        "targetTemplateId": "template-2023",
        "syncedPlaceholders": 2
      },
      {
        "targetTemplateId": "template-2025",
        "syncedPlaceholders": 2
      }
    ]
  }
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 是否全部同步成功 |
| syncedCount | number | 总共同步的占位符数量 |
| details | array | 每个目标模板的同步详情 |

### 同步匹配逻辑

```sql
-- 伪代码：同步时的匹配逻辑
UPDATE company_template_placeholder AS target
SET 
  name = source.name,
  type = source.type,
  data_source = source.data_source,
  source_sheet = source.source_sheet,
  source_field = source.source_field,
  description = source.description
FROM company_template_placeholder AS source
JOIN company_template_module AS source_module ON source.module_id = source_module.id
JOIN company_template_module AS target_module ON target_module.code = source_module.code 
                                           AND target_module.template_id = target_template_id
WHERE source.id IN (placeholderIds)
  AND source.template_id = sourceTemplateId
  AND target.template_id = targetTemplateId
  AND target.placeholder_name = source.placeholder_name
  AND target.module_id = target_module.id
```

**关键**：按 `module.code` + `placeholder_name` 匹配，不是按ID匹配

## 反向生成时的数据处理

当调用 `reverseGenerate` 接口生成子模板时，需要同时：

1. **创建模块记录**：解析历史报告，生成 `company_template_module` 记录
2. **创建占位符记录**：解析出的占位符，生成 `company_template_placeholder` 记录
3. **模块匹配**：如果可能，尝试将解析出的模块与标准模板模块关联（通过code匹配）

### 反向生成流程

```
1. 接收历史报告文件
2. 解析Word文档，提取模块结构和占位符
3. 创建子模板记录 (company_template)
4. 创建模块记录 (company_template_module)
   - 如果模块code匹配标准模板，可以复用标准模板的code
   - 如果不匹配，生成新的唯一code
5. 创建占位符记录 (company_template_placeholder)
   - 设置 status = 'uncertain'
   - 设置 confirmed_type = null
   - 其他字段使用解析结果或默认值
6. 返回子模板ID，前端进入编辑确认流程
```

## 数据库迁移脚本

### 1. 创建 company_template_module 表

```sql
CREATE TABLE company_template_module (
    id VARCHAR(64) PRIMARY KEY COMMENT '模块ID',
    template_id VARCHAR(64) NOT NULL COMMENT '所属子模板ID',
    name VARCHAR(255) NOT NULL COMMENT '模块名称',
    code VARCHAR(100) NOT NULL COMMENT '模块代码',
    description TEXT COMMENT '模块描述',
    sort INT DEFAULT 0 COMMENT '排序序号',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_template_id (template_id),
    INDEX idx_code (code),
    FOREIGN KEY (template_id) REFERENCES company_template(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='企业子模板模块表';
```

### 2. 扩展 company_template_placeholder 表

```sql
-- 添加 module_id 外键字段
ALTER TABLE company_template_placeholder 
ADD COLUMN module_id VARCHAR(64) COMMENT '所属模块ID' AFTER template_id,
ADD INDEX idx_module_id (module_id),
ADD FOREIGN KEY (module_id) REFERENCES company_template_module(id) ON DELETE CASCADE;

-- 添加管理字段
ALTER TABLE company_template_placeholder 
ADD COLUMN name VARCHAR(255) COMMENT '显示名称' AFTER placeholder_name,
ADD COLUMN type VARCHAR(20) COMMENT '类型：text/table/chart/image/ignore' AFTER name,
ADD COLUMN data_source VARCHAR(255) COMMENT '数据源' AFTER type,
ADD COLUMN source_sheet VARCHAR(255) COMMENT '来源Sheet' AFTER data_source,
ADD COLUMN source_field VARCHAR(255) COMMENT '来源字段' AFTER source_sheet,
ADD COLUMN description TEXT COMMENT '说明' AFTER source_field,
ADD COLUMN sort INT DEFAULT 0 COMMENT '排序序号' AFTER description;
```

## 注意事项

1. **权限控制**：所有接口需要验证用户是否有权限操作该企业的子模板
2. **数据隔离**：确保用户只能访问自己企业的子模板数据
3. **同步安全**：sync-placeholders 接口要确保目标模板与源模板属于同一企业
4. **占位符匹配**：同步时按 `module.code` + `placeholder_name` 字段匹配，不是按ID匹配
5. **反向生成**：reverseGenerate 接口需要同时创建模块和占位符记录
