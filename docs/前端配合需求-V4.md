# 前端配合需求文档 V4

## 背景

后端反向引擎（ReverseTemplateEngine）已完成技术改造，实现了 Sheet 类型自动识别、分级替换策略、BVD 精准行扫描等功能。前端需要适配新的返回数据结构和交互需求。

---

## 一、接口变更说明

### 1.1 反向生成接口

**接口**: `POST /api/company-template/reverse-generate`

**响应变更**:

```json
{
  "code": 200,
  "message": "反向生成完成",
  "data": {
    "template": {...},
    "matchedCount": 25,
    "pendingConfirmList": [...],
    "unmatchedLongTextEntries": [  // 新增字段
      {
        "placeholderName": "行业情况-B5",
        "value": "该行业是国民经济重要支柱产业，近年来保持较快增长...",
        "dataSource": "list",
        "sourceSheet": "行业情况",
        "sourceField": "B5",
        "longText": true
      }
    ],
    "async": false,
    "engine": "fallback"
  }
}
```

### 1.2 占位符数据结构变更

| 字段 | 类型 | 说明 |
|------|------|------|
| `dataSource` | string | 数据来源：`list`(清单) / `bvd`(BVD数据) |
| `sourceSheet` | string | 来源 Sheet 名称 |
| `sourceField` | string | 来源单元格地址，如 `B3` |
| `status` | string | 状态：`confirmed`(已确认) / `uncertain`(待确认) |

## 二、前端改造需求

### 需求1：未匹配长文本提示（P0）

**场景**: 反向生成后，部分长文本内容在 Word 中未找到匹配位置

**交互要求**:

1. **检测逻辑**
   - 调用反向生成接口后，检查 `unmatchedLongTextEntries` 字段
   - 如果列表非空，显示提示弹窗或横幅

2. **提示内容**
   ```
   以下长文本内容未在文档中自动匹配到，请手动检查：
   - 行业情况-B5（原文前50字...）
   - 集团背景情况-A3（原文前50字...）
   ```

3. **用户操作选项**
   - **忽略**: 继续后续流程
   - **手动替换**: 跳转到 OnlyOffice 编辑器，让用户手动查找替换
   - **标记为已完成**: 记录到本地，下次不再提示

**UI参考**:

## 三、接口调用时序

```
1. 用户上传历史报告 + 选择年度
          ↓
2. POST /reverse-generate
          ↓
3. 检查响应中的 unmatchedLongTextEntries
   ├─ 有未匹配项 → 显示提示弹窗
   └─ 无未匹配项 → 直接进入确认页
          ↓
4. GET /{id}/placeholders 获取占位符列表
          ↓
5. 展示占位符确认界面（分组展示）
          ↓
6. 用户确认后 POST /{id}/confirm-placeholders
          ↓
7. 完成，进入子模板编辑/下载
```

## 五、兼容性处理

### 5.1 旧数据兼容

- 旧数据可能没有 `dataSource` 字段，默认为 `list`
- 旧数据占位符命名可能不同，正常展示即可

### 5.2 降级处理

- 如果后端返回数据缺少新字段，前端静默忽略，不影响原有功能

#### 核心 API 调用

```javascript
// 扫描文档中的占位符
function scanPlaceholders() {
  const placeholders = [];
  const doc = docEditor.getDocument();
  
  doc.paragraphs.forEach((para, pIndex) => {
    para.runs.forEach((run, rIndex) => {
      const text = run.text;
      const matches = text.match(/\{\{([^}]+)\}\}/g);
      if (matches) {
        matches.forEach(match => {
          placeholders.push({
            name: match.replace(/[{}]/g, ''),
            paragraphIndex: pIndex,
            runIndex: rIndex,
            offset: text.indexOf(match)
          });
        });
      }
    });
  });
  
  return placeholders;
}

// 添加高亮
function highlightPlaceholder(placeholder) {
  docEditor.selectRange({
    paragraphIndex: placeholder.paragraphIndex,
    runIndex: placeholder.runIndex,
    offset: placeholder.offset,
    length: placeholder.name.length + 4  // 加上 {{ }}
  });
  
  docEditor.applyStyle({
    backgroundColor: '#e6f7ff',
    borderBottom: '2px solid #1890ff'
  });
}
```

### 8.5 接口调用时序（在线编辑场景）

```

        ↓

        ↓

        ↓
```
